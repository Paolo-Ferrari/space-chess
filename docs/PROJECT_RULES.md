# PROJECT_RULES.md

# Правила разработки

Версия документа: 0.2.0

---

## Назначение

Короткие правила для Space Chess.

Механики только из `GAME_DESIGN.md`. Если неясно — спросить, не выдумывать.

---

## Приоритеты

1. Работающий играбельный цикл.
2. Понятный код.
3. Простая структура.
4. Скорость — после ясности.

Не строить «AAA-архитектуру» заранее.

---

## Перед кодом

Кратко ответить:

- зачем это;
- что входит / не входит;
- какие данные нужны.

Потом писать.

---

## Код

- Один файл — одна зона ответственности.
- Не раздувать файлы без нужды.
- Не писать «на будущее», если это не используется сейчас.
- Не тащить старую RTS-логику (ресурсы, Factory, модули) в текущую игру.

Слои:

- `domain/` — правила и типы;
- `data/catalog/` — статические данные;
- `services/` — движок / хранилище;
- `ui/` — экраны и компоненты.

---

## React

- Компонент = один UI-блок.
- Игровая логика — вне «толстых» UI-компонентов.
- Состояние матча — в одном понятном месте (`matchEngine` + экран матча).
