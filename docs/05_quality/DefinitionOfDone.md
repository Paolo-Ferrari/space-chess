## 1. Цель документа

Когда фича считается готовой для merge / релиза.

## 2. Чеклист (минимум)

- [ ] Поведение описано в профильном doc (или обновлён doc в том же PR)
- [ ] Нет выдуманных механик/чисел вне Balance/Units
- [ ] Код соответствует Coding Standards
- [ ] Есть тесты на затронутые правила (если менялся domain)
- [ ] Проверена локальная сборка (`build`)
- [ ] UI не ломает утверждённый UI Flow без записи в docs

## 3. Режим обновления

Редко — при смене процесса команды.
